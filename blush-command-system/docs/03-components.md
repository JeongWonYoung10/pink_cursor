# Components

## Primitive Components

- Button: primary, gradient, secondary, quiet, recovery
- Card: glass, blush, gradient, technical
- Badge: status, shortcut, category, risk, success
- Input: search, prompt, inline-agent
- Progress: linear, checklist, stepper, queue
- Toast: success, warning, recovery, agent-status

## Product Components

- AgentBubble
- CommandRow
- DiffLine
- FileTreeItem
- EditorTab
- StatusPill

## Rules

- One strong primary action per screen.
- Agent commands use Sparkles/Bot visual language.
- Diff and technical surfaces prioritize readability over decoration.
